# agent-ready/core вЂ” Design Spec

**Date:** 2026-06-04
**Status:** Approved
**Scope:** `packages/core` вЂ” static file generators for AI Agent Readiness

---

## Context

`agent-ready` is an npm monorepo that helps Next.js/React/Vite apps pass the checks evaluated by isitagentready.com (Cloudflare). This spec covers Phase 1: the core package with pure TypeScript generators.

**Planned packages after core:**
- `packages/next` вЂ” Next.js plugin
- `packages/cli` вЂ” `agent-ready init` + `agent-ready audit`

---

## Architecture

The core package is a pure TypeScript library: no I/O, no network, no external dependencies. Every generator is a deterministic function that takes a validated config and returns an array of `GeneratedFile` objects. The caller (CLI, Next.js plugin, etc.) is responsible for writing files to disk.

```
AgentReadyConfig
    в”‚
    в–ј
validate(config)          в†ђ throws ZodError-like errors with human messages
    в”‚
    в–ј
generate*(config)         в†ђ 6 generators, each returning GeneratedFile[]
    в”‚
    в–ј
GeneratedFile[]           в†ђ { path: string; content: string }[]
```

**Public exports:**
- `defineConfig(config)` вЂ” identity function for TypeScript type inference (Vite pattern)
- `generateAll(config)` вЂ” runs all enabled generators, returns merged `GeneratedFile[]`
- Individual generators for granular use: `generateRobots`, `generateLlmsTxt`, `generateSitemap`, `generateMcpServerCard`, `generateAgentSkills`, `generateApiCatalog`
- `AgentReadyConfig` type and sub-types

## Config Interface

```ts
export interface AgentReadyConfig {
  site: {
    name: string
    description: string
    baseUrl: string          // e.g. "https://example.com" вЂ” no trailing slash
    language?: string        // BCP 47, default "en"
  }

  content: {
    llmsTxt?: boolean                 // default true
    llmsFullTxt?: boolean             // default false
    markdownNegotiation?: boolean     // hint for future HTTP middleware
    links?: Array<{
      title: string
      url: string
      description?: string
    }>
  }

  bots: {
    aiTrain?: boolean        // Content-Signal: ai-train (allow AI training crawlers)
    aiInput?: boolean        // Content-Signal: ai-input (allow AI inference crawlers)
    search?: boolean         // Content-Signal: search (allow search crawlers)
    allowedBots?: string[]   // user-agent strings to explicitly allow
    blockedBots?: string[]   // user-agent strings to explicitly block
  }

  protocol: {
    mcpServerCard?: boolean | MCPServerCardOptions
    agentSkills?: boolean
    apiCatalog?: boolean | APICatalogOptions
    oauth?: boolean          // reserved for future use
  }

  commerce?: boolean         // reserved вЂ” may affect agent-skills output
}

export interface MCPServerCardOptions {
  name?: string              // defaults to site.name
  description?: string       // defaults to site.description
  capabilities?: string[]    // e.g. ["tools", "resources", "prompts"]
  authType?: "none" | "oauth2" | "api-key"
}

export interface APICatalogOptions {
  apis: Array<{
    title: string
    description?: string
    openapi?: string         // URL to OpenAPI spec
  }>
}
```

## Generators

### Universal contract

```ts
type GeneratedFile = { path: string; content: string }
type Generator<T = void> = (config: AgentReadyConfig, options?: T) => GeneratedFile[]
```

All generators skip output (return `[]`) when their corresponding config flag is `false` or `undefined`.

---

### `robots.ts` в†’ `robots.txt`

Highest impact on isitagentready.com score.

**Output format:**
```
# Generated by agent-ready
# Content-Signal directives (Cloudflare)
Content-Signal: ai-train=<allow|deny>
Content-Signal: ai-input=<allow|deny>
Content-Signal: search=<allow|deny>

Sitemap: https://example.com/sitemap.xml

# AI Training bots
User-agent: GPTBot
User-agent: CCBot
User-agent: Diffbot
...
Disallow: /   (if aiTrain=false)
Allow: /      (if aiTrain=true)

# AI Input/Inference bots
User-agent: ClaudeBot
User-agent: anthropic-ai
User-agent: Bytespider
...
Disallow: /   (if aiInput=false)
Allow: /      (if aiInput=true)

# General crawlers
User-agent: *
Allow: /
```

**Known AI bot lists (curated at implementation time):**
- AI Training: `GPTBot`, `CCBot`, `Diffbot`, `Google-Extended`, `FacebookBot`, `PerplexityBot`, `Omgilibot`, `Kangaroo Bot`, `DataForSeoBot`
- AI Input: `ClaudeBot`, `anthropic-ai`, `Bytespider`, `cohere-ai`, `AI2Bot`, `Amazonbot`
- Custom allow/block lists are appended as separate `User-agent` blocks

---

### `llms-txt.ts` в†’ `/llms.txt` (+ optionally `/llms-full.txt`)

Follows the [llmstxt.org spec](https://llmstxt.org).

**`llms.txt` format:**
```markdown
# {site.name}

> {site.description}

## Links

- [Title](url): Description
- [Title](url): Description
```

**`llms-full.txt`** вЂ” same structure but includes fetched/expanded content of each link. Since core has no I/O, `llms-full.txt` is generated with the same content as `llms.txt` unless the caller provides pre-fetched content via the config links' optional `content` field (to be added in a future iteration). For now, `llms-full.txt` = `llms.txt` content.

---

### `sitemap.ts` в†’ `/sitemap.xml`

Minimal valid sitemap вЂ” primarily used as the `Sitemap:` pointer in `robots.txt`.

```xml
<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <url>
    <loc>https://example.com/</loc>
    <lastmod>YYYY-MM-DD</lastmod>
    <changefreq>weekly</changefreq>
    <priority>1.0</priority>
  </url>
</urlset>
```

`lastmod` is passed in via config or defaults to the date at generation time (caller passes it explicitly to keep the generator deterministic in tests).

---

### `mcp-server-card.ts` в†’ `/.well-known/mcp/server-card.json`

Follows the [modelcontextprotocol.io server card draft](https://modelcontextprotocol.io).

```json
{
  "name": "Example Site",
  "description": "...",
  "url": "https://example.com",
  "version": "1.0.0",
  "capabilities": ["tools"],
  "auth": {
    "type": "none"
  }
}
```

Only generated when `protocol.mcpServerCard` is `true` or an `MCPServerCardOptions` object.

---

### `agent-skills.ts` в†’ `/.well-known/agent-skills/index.json`

Follows Cloudflare's agent-skills-discovery RFC.

```json
{
  "version": "1",
  "skills": []
}
```

When `commerce: true`, a default e-commerce skill set is added. Generated when `protocol.agentSkills` is `true`.

---

### `api-catalog.ts` в†’ `/.well-known/api-catalog`

Follows [RFC 9727](https://www.rfc-editor.org/rfc/rfc9727).

```json
{
  "apis": [
    {
      "title": "My API",
      "description": "...",
      "openapi": "https://example.com/openapi.json"
    }
  ]
}
```

Only generated when `protocol.apiCatalog` is `true` or an `APICatalogOptions` object.

---

## Validation (`validate.ts`)

`validate(config)` throws a structured `AgentReadyValidationError` with a human-readable message array if:
- `site.baseUrl` is missing or not a valid URL
- `site.name` or `site.description` is empty
- `protocol.apiCatalog` is an options object with no `apis` entries

Returns `void` on success вЂ” does not transform the config.

## Testing Strategy

Each generator test file:
1. Calls the generator with a minimal valid config
2. Asserts the correct `path` value
3. Asserts `content` contains required strings/structure
4. Tests edge cases: flags disabled (returns `[]`), custom bot lists, optional fields

Tests are deterministic вЂ” no `Date.now()` calls inside generators. Date is passed via config or a parameter.
